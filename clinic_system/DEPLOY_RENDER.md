# 🚀 رفع نظام العيادة أونلاين — دليل خطوة بخطوة (Render.com)

هذا الدليل يوضح كيفية رفع السيستم على رابط دائم يشتغل 24 ساعة، بدون الحاجة
لترك جهازك الشخصي شغال. الاستضافة **مجانية** ومناسبة للاستخدام الداخلي
(العيادة + شئون الطلاب).

---

## ⚠️ ملاحظة مهمة عن قاعدة البيانات

الخطة المجانية في Render بتمسح أي ملفات بتتغيّر (زي `clinic.db`) كل مرة
السيرفر يعيد التشغيل (وده بيحصل تلقائياً بعد فترة عدم استخدام). يعني:
- **بيانات الطلاب/الموظفين/الأدوية** (ملفات Excel) — هتفضل موجودة لأنها مرفوعة مع الكود
- **الزيارات والتقارير اللي تتسجل من الاستخدام الفعلي** — ممكن تتمسح بعد فترة

**الحل المقترح:** لو عايزة البيانات تفضل محفوظة بشكل دائم 100%،
بنحتاج نفعّل "Persistent Disk" (مدفوعة بسعر رمزي تقريباً $1/شهر) أو ننقل
قاعدة البيانات لخدمة مجانية تانية زي **Render PostgreSQL** المجانية.
لو حابة، قوليلي وأظبط الكود لذلك. الدليل ده بيغطي النسخة الأبسط أولاً.

## الخطوات

### 1. إنشاء حساب GitHub (لو مش موجود)
- روحي على **github.com** واعملي حساب مجاني
- ده المكان اللي الكود هيتخزن فيه قبل الرفع

### 2. رفع الكود على GitHub
- افتحي **github.com/new** لإنشاء "Repository" جديد
  - الاسم: `hti-clinic-system`
  - اختاري **Private** (مهم — البيانات حساسة)
  - اضغطي **Create repository**
- بعد الإنشاء، هتظهر تعليمات لرفع ملفات. اتبعي قسم
  **"...or push an existing repository from the command line"**
  من جهازك (محتاجة Git مثبت):
  ```bash
  cd clinic_system
  git init
  git add .
  git commit -m "أول رفعة للسيستم"
  git branch -M main
  git remote add origin https://github.com/<اسم_حسابك>/hti-clinic-system.git
  git push -u origin main
  ```

> 💡 **مهم جداً:** لا ترفعي ملف `.env` على GitHub أبداً لأنه فيه باسورد حقيقي.
> تأكدي إنه مكتوب في ملف `.gitignore` (موجود جاهز ضمن المشروع).

### 3. إنشاء حساب Render
- روحي على **render.com** وسجّلي بحساب GitHub بتاعك (أسهل وأسرع)

### 4. إنشاء "Web Service" جديد
- من Render Dashboard: **New** ← **Web Service**
- اختاري الـ Repository اللي رفعتيه (`hti-clinic-system`)
- الإعدادات:
  - **Name:** `hti-clinic-system` (أو أي اسم تحبيه — هيظهر في الرابط)
  - **Region:** أقرب منطقة (Frankfurt أو أي خيار متاح)
  - **Branch:** `main`
  - **Runtime:** Python 3
  - **Build Command:** `pip install -r requirements.txt`
  - **Start Command:** `gunicorn app:app --bind 0.0.0.0:$PORT`
  - **Instance Type:** Free

### 5. إضافة بيانات `.env` بشكل آمن
- في نفس صفحة الإعداد، لف لتحت لقسم **Environment Variables**
- اضغطي **+ Add Environment Variable** وأضيفي كل سطر من ملف `.env`
  بشكل منفصل، مثال:

  | Key | Value |
  |---|---|
  | `ONEDRIVE_EMAIL` | `studens_affairs@hti-o.edu.eg` |
  | `ONEDRIVE_PASSWORD` | (الباسورد الحقيقي) |
  | `GRAPH_CLIENT_ID` | `ca060fcd-6c12-44e8-a9be-fd01ad108fef` |
  | `GRAPH_TENANT` | (Tenant ID بتاعك) |
  | `ONEDRIVE_FOLDER_URL` | (لينك مجلد العياده) |

### 6. الضغط على Deploy
- اضغطي **Create Web Service**
- استني 2-3 دقائق لحد ما يخلص البناء
- في الأعلى هيظهر رابط شكله:
  ```
  https://hti-clinic-system.onrender.com
  ```

### 7. تجربة الرابط
افتحي:
```
https://hti-clinic-system.onrender.com/login
```
ولازم تظهر صفحة تسجيل الدخول مباشرة.

## ملاحظة عن السرعة في الخطة المجانية

لو السيرفر مالوش زيارات لمدة طويلة (15 دقيقة تقريباً)، Render بيوقّفه تلقائياً
لتوفير الموارد. أول زيارة بعد فترة توقف بتاخد حوالي 30-50 ثانية لحد ما
يرجع يشتغل، وبعد كده بيكون سريع عادي. ده طبيعي في الخطة المجانية.

## بعد كل تحديث للكود

لو عملتي أي تعديل على السيستم بعد كده، الخطوات تكون:
```bash
git add .
git commit -m "وصف التعديل"
git push
```
Render هيكتشف التحديث ويعيد البناء تلقائياً، الرابط نفسه يفضل ثابت.
